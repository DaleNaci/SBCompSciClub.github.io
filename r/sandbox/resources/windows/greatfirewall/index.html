Task

  • There is a secret three letter word somewhere in our database (this word doesn't have to be English, and may as well be any arrangement of three letters)
  • It can't be queried conventionally
  • It has something to do with adding data to the reference "/firewall/", in that we wrote up some rules that rejects write access unless the object added contains a key "secret" where its value is the secret word we are looking for, as per the following example:
    • { "secret": "insert_the_secret_word_here" }
  • Your job is to write your name somewhere within "/firewall/"
